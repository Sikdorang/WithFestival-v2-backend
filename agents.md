# 에이전트·협업 규칙

## HTTP API와 Swagger

- **엔드포인트를 추가·수정·삭제할 때** Nest 컨트롤러/DTO만 바꾸지 말고, **같은 작업 안에서 Swagger(OpenAPI)도 맞출 것.**
- 이 프로젝트는 도메인별로 `src/swagger/{auth,stores,menus,…}/`에 데코레이터 조합·`dto.openapi.ts`·태그 상수를 둡니다. 해당 도메인 파일을 갱신하고, 컨트롤러에는 그룹 데코레이터(예: `ApiXxxDocs`)를 연결합니다.
- **체크**: 경로·메서드·쿼리/경로 파라미터·본문 스키마(multipart 포함)·주요 응답 코드(401/404/400 등)·요약·설명이 실제 동작과 일치하는지 확인합니다.

※ “자동”은 빌드 파이프가 알아서 고쳐 주는 의미가 아니라, **구현 변경 시 문서 변경을 빼먹지 않는다**는 뜻으로 따릅니다.
